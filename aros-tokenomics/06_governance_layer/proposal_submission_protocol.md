# Proposal Submission Protocol

This protocol standardises how proposals enter the governance pipeline.

## Requirements

- Detailed description with objectives, scope, and stakeholder impact.
- Quantitative analysis (financial, technical, or compliance) supporting the proposal.
- Implementation plan including resource needs and timeline.
- Risk assessment with mitigation strategies.

## Submission Flow

1. **Drafting**: Proposer completes template and collects endorsements.
2. **Intake Review**: Governance secretariat checks completeness and assigns committees.
3. **Feedback Cycle**: Committees provide feedback; proposer updates accordingly.
4. **Scheduling**: Proposal scheduled for voting window with public notice.

## Tools

Proposals tracked in shared project management system linked to Notion and GitHub issues for
traceability. All documents archived in immutable storage.
